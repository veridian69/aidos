# Migration v1.3.0 → v1.4.0

## Summary

This migration adds a new **Auditor Notes** section to every artifact template (Problem, Solution, Tech Design, Testing) at all scales. The Auditor Notes section is the persistent home for findings from the AIDOS Auditor — a structural change introduced to support autonomous build–audit–fix loops where sub-agents cannot read each other's chat history.

No content is moved, renamed, or retired. The change is purely additive: a new section appended at the bottom of each artifact, after the existing Issues / Decisions tables.

A new Breakdown rubric (`src/rubrics/breakdown.md`) is also introduced for auditing Epic-scope decomposition into Features and Stories. This rubric is loaded by the Auditor skill in addition to the existing rubric set, but does not change how existing artifacts are written. No artifact migration is required for the new rubric.

## File renames

None.

## Content changes

### All four artifact types (Problem, Solution, Tech Design, Testing)

Append the following section at the END of the file, AFTER the existing `## Decisions` table (and any other trailing section). Do not modify any existing section.

```markdown

## Auditor Notes

<!--
Populated by the AIDOS Auditor skill. Rewritten on each audit pass — latest
findings only; git carries the history. Cleared once the artifact is final (no
open Bugs, no new findings on the latest pass).

Findings are classified per framework.md § Builder / Auditor Separation:
- Bug — must fix before proceeding
- Risk — surface; may become an Issue
- Idea — noted, not actioned
-->

### Bugs (open)

_None_

### Risks

_None_

### Ideas

_None_
```

For artifacts where the Auditor has already been run and produced findings outside the artifact (e.g. in chat), populate the new sections with those findings using the per-finding format:

- `- [B1] {Brief finding} — evidence: "{cited quote from artifact, or section reference}"`
- `- [R1] …`
- `- [I1] …`

## Metadata changes

In each artifact's metadata block at the top:

- Before: `**AIDOS Version:** 1.3.0`
- After: `**AIDOS Version:** 1.4.0`

No other metadata fields change.
